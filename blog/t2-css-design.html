ID vs Classes

CSS IDs and Classes

November 9, 2014

It will take an coder less than a few days of HTML and CSS to at least come to the question of how to give all the different lines of code different attributes without effecting the rest of the code. The main tags or commands in HTML are p (paragraphs), div (for boxes), table (for tables), and img (for images). They seem to be pretty simple but if you have a ton of a dozen images synced into paragraphs along-side a few boxes with tables in them your definitely going to have a hard time referring to each and every one of them in a unique way. This is where CSS's method of setting classes and ids save the day. Basically, an id is and id. It creates a specific and unique reference to the intended image, paragraph or any other element, you plan on changing in your CSS file. Classes are the same in that you can refer to whatever is set in that class by mentioning the class name you have customized for it. The main difference is when you have more than I one element that have the same CSS atrtibutes. Thats when classes stand alone and get teh job done. This way you won't have to worry about referring to, say, four different ids in your CSS file just to get them set to the same font. Instead you set a class reference and refer to them once in CSS via the class name and woala...finish.
