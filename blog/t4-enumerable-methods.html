Map vs. Each

Enumerable Methods

November 22, 2014

You might have wondered how programming languages are full of similar codes and methods. The vast majority of methods that look alike are simply identical in part of what they do, but are distinct in other aspects of operation. One of the most popular confusions that takes place in the early stages of programming is the .map and .each method.

In a quick glimse the two do the exact same thing, they iterate over a set of elements. Another similarity is that they execute a code block that could entail any kind of code. Other methods that have the same attributes are .collect and .select. Despite having these similarities the two are extremely different.

The .each method iterates over a set of elements like the numbers in an array running the block of code on each object, but refrains from returning the modified object. It returns the orignal value that was sent to the block:

[10, 20, 30].each {|x| x + 5}
# => [10, 20, 30]

On the other hand, .map does the exact same thing returning the modified version of the object as oppose to the original object:

[10, 20, 30].map {|x| x + 5}
# => [15, 25, 35]

Although this seems to imply that map is the destructive form of each. This is not true. There is a slight differenc between returning a distinct modified object of a separate original object and altering the original so that it only exists in the form of the modified object. The destructive version of .map is followed by an exclamation mark and each does not have a destructive version:

numbers = [10, 20, 30]
numbers.map {|x| x + 5}
puts numbers
# => [10, 20, 30]

numbers = [10, 20, 30]
numbers.map! {|x| x + 5}
puts numbers
# => [15, 25, 35]
