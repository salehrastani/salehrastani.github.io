The Battle of the Hash and Array

Ruby Arrays and Hashes

November 14, 2014

Today were going to go over arrays and hashes. Basically, arrays are a set of elements. These elements could be anything from strings to integers. Like all objects in Ruby arrays can be set to variables and they also have a ton of cool methods you can play around with like .join, .map, .each. A comma separates each element in an Array. Every element has a position is the array, by which it is referred to. The first reference number is 0, which refers to the first element of the array. The second element is referred to by the number 1, and so on. The reference number is placed in brackets next to the arrayâ€™s name to call the element like so:

numbers = [1, 2, 3, 4]

numbers[0] => 1

numbers[1] => 2

numbers[2] => 3

numbers[3] => 4

Despite being similar, hashes on the other hand have different attributes. One of the main differences is that hashes have key-value pairs instead of elements. However, the same way arrays are set to variable names hashes also acquire variable names. Hashes have the equal-or-greater-than sign between the key and value. A unique trait of hashes is that you can refer to the keys and the values separately. You can also refer to all of the keys or values at the same time.

menu = {

burger => 6

fries => 2

salad => 3

}

menu[burger] => 6

menu[fries] => 2

menu[salad] => 3

menu[6] => burger

menu[2] => fries

menu[3] => salad
