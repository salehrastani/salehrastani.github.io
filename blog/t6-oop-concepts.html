Modules vs Classes

Ruby Modules and Classes

December 8, 2014

A class is structure that is used to make examples, technically known as instances objects of themselves. The contents of a class enable the instances with state and behavior. State include variables, data, attributes and constants. Behavior simply includes the methods that execute a give code. For example 

    class Devbootcamp
      def initilize(student)
        students = []
        @student = student
      end
      def add
        students << @student
      end
    end

On the other hand, Modules simply encapsulate methods and constants. The module holds the code with it, which can be used in another a different scope of code by simply calling it. The unique trait of modules is that it is mostly used when a collection of code and variables are good together but do not require the creation of an instance. 
    Module Classes

      @student_cap = 30

      def within_cap
        if @student_cap <= 30
          return true
        end
      end

    end

Ultimately, the module is called within the class or method that you need it: 
    class Devbootcamp
      def in

        students = []
        @student = student
      end
      def add
        students << @student
      end
    end
